# Blueprint Assessment Delivery

Processes completed Blueprint AI readiness assessments and generates client-ready deliverables.

## When to Use

- A prospect has completed a Blueprint assessment form
- Generating the scored report with AI use case framework
- Calculating EBITDA impact ranges for identified opportunities
- Preparing the presentation for the client readout meeting

## Workflow

### 1. Intake
- Confirm assessment data is complete (company info, current tech stack, pain points, strategic goals)
- Flag any missing fields before proceeding
- Tag the lead in Pipeline board: `blueprint-qualified`

### 2. Scoring
- Score each dimension of AI readiness (data maturity, infrastructure, talent, governance, use case clarity)
- Apply the weighted scoring framework (reference: Blueprint scoring model)
- Generate overall readiness tier: Foundation / Emerging / Advanced / Leading

### 3. Use Case Identification
- Map identified pain points to AI use case categories
- Rank use cases by: feasibility × business impact × data readiness
- Generate top 3-5 recommended use cases with implementation complexity estimate

### 4. EBITDA Impact Analysis
- For each recommended use case, calculate estimated EBITDA impact range
- Use industry benchmarks and company-specific revenue/cost data
- Present as conservative / moderate / aggressive scenarios

### 5. Report Generation
- Compile into client-branded report (reference: Blueprint report template)
- Include: executive summary, readiness scores, use case roadmap, EBITDA projections, recommended next steps
- Recommended next steps should naturally lead to Blue Orange engagement

### 6. Delivery Prep
- Create meeting prep doc for the readout session
- Prepare talking points for each section
- Flag any sensitive findings that need careful positioning

## Output Artifacts
- `blueprint-report-[company].pdf` — Client-facing assessment report
- `blueprint-prep-[company].md` — Internal meeting prep notes
- Pipeline task updated with score and recommended next steps

## Approval Gate
Report must be reviewed by Josh or Colin before client delivery.
